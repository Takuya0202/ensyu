# プロダクト概要

## 1. 誰の困りごとを解決するか
- IT知識がないものの、ソフトウェアを作ってみたい意欲がある人
- IT知識があり、プロダクトを作ることが好きだが、なかなか事業案を思い出せない人
- 学生向けの就活などの実績を作りたい人

## 2. 最低限の機能（MVP）
- プロダクト案を投稿して募集する機能
- 案に対して参加申請をする機能
- トーク機能（DM、チーム）

## 3. なぜ自分がそれを作るのか
- 自分自身も最近作ってみたいプロダクトが思いつかず、このようなサービスがあればやってみたいと思ったため
- AIエージェントの発達により、作る人のハードルが低くなっており需要があると考えたため
- 実際にSNS上でもバイブコーディングなど個人開発投稿が増えているため

## 4. 後回しする機能
- 実績機能
- いいね機能（申請はしない）
- 募集要項の充実

## 5. 備考・現時点での困りごと
- 本サービスを実際にリリースするかは任意とし、ガクチカの一部として使えることを重視する
- コミュニティ規約をどう調整していくかが課題

## 6. 技術選定

### frontend
- 言語: TypeScript
- フレームワーク: Next.js (App Router)
- ホスティング: Vercel
- UIライブラリ: shadcn または自前
- 状態管理: Zustand

### backend
- 言語: TypeScript
- フレームワーク: Hono
- ORマッパー: Drizzle
- 認証: Clerk
- ホスティング: Cloudflare Workers
- DB: Cloudflare D1
- ストレージ: Cloudflare R2

### 共通
- monorepo 構成
- バリデーション: Zod
- パッケージマネージャー: pnpm
