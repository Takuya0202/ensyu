# 認証設計方針

## 採用技術

**better-auth** を使って認証を実装する。

## 方針

- 認証ロジックは自前で実装せず、better-auth に委譲する
- セッション管理・トークン検証も better-auth が担う
- メール/パスワード認証と OAuth（Google / GitHub）をサポートする
- OAuth ログイン時、未登録ユーザーは自動的に登録される

## フロントエンド側

- better-auth のクライアントSDK を使用する
- 未認証ユーザーへのアクセス制御は Next.js ミドルウェアで行う

## バックエンド側

- Hono ミドルウェアで better-auth のセッションを検証する
- 検証済みのユーザー情報はコンテキスト経由でルートハンドラに渡す

## 認証フロー

イベントストーミング図を参照: `docs/product/event-storming-legend.drawio`（認証フローページ）

| フロー | 概要 |
|--------|------|
| 新規登録 | メール/パスワードで登録 → セッション作成 |
| ログイン | 認証成功でセッション作成、失敗はエラー返却 |
| OAuthログイン | Google/GitHub で認証 → 未登録なら自動登録 → セッション作成 |
| ログアウト | セッション削除 |
