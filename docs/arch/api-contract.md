# API コントラクト設計方針

## 方針

フロントエンドとバックエンドの通信は **Hono RPC** を使い、型を共有する。

## 仕組み

1. バックエンド（`package/api`）で Hono のルーターを定義し、型をエクスポートする
2. フロントエンド（`package/web`）はその型を import して `hc`（Hono Client）経由でリクエストを送る
3. リクエスト・レスポンスの型は自動で推論されるため、手動の型定義は不要

## バリデーション

- リクエストボディ・クエリパラメータは Zod で定義し、Hono の `zValidator` でバリデートする
- Zod スキーマはフロントエンドでも再利用してフォームバリデーションに使う
