# ユーザーフロー

## 前提

MVP スコープのフローのみ記載する。

---

## フロー 1: プロダクト案を投稿する

```
[ログイン済みユーザー]
  └─> 投稿フォームを開く
        ├─ タイトル（必須）
        ├─ 概要・説明（必須）
        ├─ 稼働期間（任意）
        └─> ポジションを追加（1つ以上必須）
              ├─ カテゴリ（フロントエンド / バックエンド / デザイナー 等）
              ├─ 技術スタックタグ（任意）
              └─ 求める人物像タグ（任意）
  └─> 投稿する
        → 案が一覧に表示される
        → プロジェクトのチームトークチャンネルが自動生成される（投稿者のみ）
```

---

## フロー 2: 案に参加申請する

```
[ログイン済みユーザー]
  └─> 投稿一覧を見る（ゲストも閲覧可）
  └─> 気になる案の詳細ページへ
  └─> 希望するポジションの申請ボタンを押す
  └─> 申請メッセージを入力して送信する
  └─> 投稿者に通知が届く（申請管理画面へのリンク付き）
  └─> 投稿者が申請管理画面で申請メッセージを確認する
  └─> 投稿者が承認 / 却下を選択する
        ├─ 承認の場合
        │    └─> 申請者に承認通知が届く
        │    └─> 申請者がチームトークに自動追加される
        └─ 却下の場合
              └─> 申請者に却下通知が届く（理由なし）
```

---

## フロー 3: DM でやり取りする

```
[ログイン済みユーザー A → ユーザー B]
  └─> B のプロフィール or 投稿詳細から DM を開始
  └─> メッセージを送受信する
  ※ ログイン済みであれば誰とでも DM できる
```

---

## フロー 4: チームトークを使う

```
[プロジェクトメンバー全員]
  └─> プロジェクト投稿時にチャンネルが自動生成される
  └─> 投稿者は投稿直後から発言できる（1人でもOK）
  └─> 申請が承認されるたびに承認者がチャンネルに自動追加される
  └─> チームチャンネルでメッセージを送受信する
```

---

## フロー 5: 投稿を締め切る

```
[投稿者]
  └─> 投稿詳細画面で「募集終了」ボタンを押す
  └─> 投稿のステータスが「締切」に変わる
  ※ ポジション単位でも個別に締め切れる
  ※ 締切操作は手動のみ（自動締切はMVP対象外）
```

---

## 画面一覧

| 画面 | 説明 |
|------|------|
| ホーム / 案一覧 | プロダクト案の一覧（デフォルト新着順 / 気になる順ソート可） |
| 投稿詳細 | 案の詳細・ポジション一覧・参加申請ボタン |
| 投稿作成 / 編集 | 新規案の作成・既存案の編集 |
| マイページ | 自分の情報・投稿・参加プロジェクト・気になり一覧 |
| 他者プロフィール | 他ユーザーの情報・投稿・参加プロジェクト |
| 申請管理 | 受けた申請の一覧・承認 / 却下 |
| DM | 1対1のメッセージ |
| チームトーク | プロジェクトメンバー全員のチャット |
| 通知 | 申請受信・承認・却下の通知一覧 |
| ログイン | Google OAuth 認証 |
